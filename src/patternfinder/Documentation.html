patternfinder

This package contains the classes which are used to create the patternstring used by the calculator and contains default factories like DecimalFactory, WordFactory and etc. The class PatternFinder is what is used to create an PatternString and is called by the calculator to create/calculate a string.

Factories

What they do

The factories will go through a patternstring and do appropriate actions based on the factory. For example, WordFactory will create patterns of the word pattern.

When they are called/used?

The factories are called atleast once, and the order of the factories called depends on the order of which they are added to the lists PatternFinder.factories and PatternFinder.extensions. The factories in the list PatternFinder.factories are called first. If the Factory return an instance of patternfinder.pattern.factories.Result with the "tag" didEditPattern true, all the factories will be iterated again, stopping the current iteration.

Note

Note that specialWordFinder is a "standard" factory, called just before the factories of the list PatternFinder.extensions!
